# Interview UX Improvement Plan

## 问题诊断

### 问题1：问题转换生硬，无回应

**用户反馈**：洋洋洒洒写了一大段，结果机器人直接跳到下一个问题，完全没有回应

**日志分析**：

```
用户回答: 125字符的长回答（关于25年工作经历）
LLM返回: "是" (需要追问)
LLM返回: "否" (生成追问时判断为不需要)
直接跳到: "您在工作中遇到过最大的挑战是什么？"
```

**根因**：

1. `isFollowupNeeded` 判断需要追问
2. `generateFollowup` 再次判断，回答被判定为"充分" → 返回 SKIP
3. **没有任何中间过渡，直接跳到下一个问题**

**当前逻辑是二元的**（追问 or 跳转），没有"回应+追问"或"回应+过渡"的中间状态。

---

### 问题2：访谈结束延迟

**用户反馈**：等待很久，然后告知访谈结束，体验不好

**日志分析**：

```
17:55:10 - 用户回答最后一个问题
17:55:12 - "开始分析"
17:55:13 - LLM调用生成报告
17:55:49 - 报告生成完成 (耗时36秒!)
17:55:49 - 发送"访谈已完成"
```

**根因**：

- 报告生成是**同步阻塞**的，用户等待36秒才收到最终消息
- 应该先发送"正在生成报告"的提示，再异步生成报告

---

## 优化方案

### 方案1：增加回应/过渡机制（推荐）

**核心思路**：在用户回答后，至少发送一个简短的回应或确认，再继续下一步

**改动点**：

1. **修改 interviewing.ts** - 在跳转下一问题前，先发送一个确认性的过渡消息：

```typescript
// 方案A: 简单确认
if (shouldProceedToNext) {
  // 添加一个简短的确认/总结
  const acknowledgment = await generateAcknowledgment(
    userAnswer,
    currentQuestion,
  );
  if (acknowledgment) {
    // 先发送确认/总结
    sendReply(acknowledgment);
    // 然后发送下一个问题（在下一条消息中）
  }
}
```

2. **修改 prompt.service.ts** - 新增确认消息模板：

```typescript
generateAcknowledgment: {
  template: `用户刚刚回答了一个关于"{{topic}}"的问题。

用户回答: {{userAnswer}}

请生成一个简短的确认/回应（20-40字），要求：
1. 简要肯定用户的回答
2. 可以提及回答中的关键点
3. 语言亲切自然
4. 不要直接问下一个问题

直接输出回应内容。`,
  variables: ['topic', 'userAnswer'],
}
```

**效果**：

- 用户回答 → 收到"感谢分享您在浩鲸25年的经历..." → 收到下一个问题
- 两步分离，体验更好

---

### 方案2：报告生成异步化

**核心思路**：访谈结束时先发送提示，异步生成报告

**改动点**：

1. **修改 analyzing.ts / graph.ts** - 在开始生成报告前，先发送提示：

```typescript
// 1. 先立即发送提示
await sendReply("感谢您的分享！访谈已结束，正在生成报告，请稍候...");

// 2. 异步生成报告（不阻塞回复）
setTimeout(async () => {
  const report = await generateReport(...);
  await saveReport(report);
  // 可选：发送报告生成完成的通知
}, 0);
```

2. **或者更简单的方案**：不等待报告生成完成，直接返回"访谈已完成"，报告在后台异步生成

**效果**：

- 用户回答最后一个问题 → 立即收到"访谈已完成，感谢参与！"
- 报告在后台异步生成，不阻塞用户

---

## 推荐方案

**组合使用**：

| 问题         | 方案              | 改动量 |
| ------------ | ----------------- | ------ |
| 问题转换生硬 | 增加确认/过渡消息 | 小-中  |
| 结束等待久   | 异步生成报告      | 小     |

### 改动清单

1. **prompt.service.ts** - 新增 `generateAcknowledgment` 模板
2. **followup.service.ts** - 新增 `generateAcknowledgment` 函数
3. **interviewing.ts** - 在跳转下一问题前调用确认生成
4. **analyzing.ts** - 访谈结束时先发送提示，再异步生成报告

---

## 风险与缓解

| 风险         | 影响                   | 缓解                       |
| ------------ | ---------------------- | -------------------------- |
| 确认消息过多 | 每轮多一次LLM调用      | 可设置为只对长回答生成确认 |
| 异步报告丢失 | 报告未生成但用户已离开 | 报告持久化，后台可重试     |
| 两次回复顺序 | 可能乱序               | 使用队列或确认机制         |

---

**改动量估算**：

- 方案1（确认机制）: ~30行代码，2个新函数/模板
- 方案2（异步报告）: ~10行代码改动
