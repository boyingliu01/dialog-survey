## TypeScript Conventions

### Naming
| Type | Convention | Example |
|------|-----------|---------|
| Classes | PascalCase | `ConversationEngine` |
| Interfaces | PascalCase | `ProcessMessageInput` |
| Functions/Vars | camelCase | `processMessage`, `userId` |
| Private | `private` keyword | `private async loadOrCreateState()` |

### Imports
- Relative imports **require `.js` extension** (ESM): `import { foo } from "../bar.js"`
- `import type` for type-only imports
- Named imports from npm

### Type Rules
- Explicit return types on public functions
- Interfaces for object shapes, type for unions
- Zod schemas for runtime validation (`z.infer` for TS type)
- `noUnusedLocals`, `noUnusedParameters` enforced by tsconfig

### Error Handling
- Custom error classes (`StatePersistenceError`, etc.)
- `instanceof Error` narrowing, never `any`
- Biome: `noConsole: error`, `noExplicitAny: warn`
- Use structured logger: `import { info, error, warn } from "../utils/logger.js"`
